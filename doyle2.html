‣ Doyle: Part 2
This project is an extention of my project Doyle: Part 1. My goal was to explore how to create scalable designs, understand design principles in depth, and also learn tools to create a powerful component library.
Timeline: Spring 2020, Side project
Contributors: Soyee
Final
Research
Before auditing and reflecting on my own designs, I kicked off this project by looking at other successful design system to understand common use cases for compontents.
Key insights
  1. Small details create meaningful changes to the design language as a whole
  2. Prioritize consistency and simplicity when creating scalable designs
  3. Facilitate collaboration between developers and designers
Define
Problem statement: Undefined design language creates inefficiency, insecurity and inconsistency for developers and designers
Goal: How might we create a consistent design language that is accessible, scalable, and transparent?
Success metrics: KPIs
  1. Adoption: Count of CSS classes, component names, or imports over time
  2. Coverage: Lower count of deprecated components is better
