Part 2: Design System

This project is an extention of the project seen above. My goal was to explore how to create scalable designs, understand design principles in depth, and also learn tools to create a powerful component library.

Problem space
Undefined design language creates inefficiency, insecurity and inconsistency for developers and designers
Goal
How might we create a consistent design language that is accessible, scalable, and transparent?
Sucess metrics: KPI's
  1. Adoption: Count of CSS classes, component names, or imports over time
  2. Coverage: Lower count of deprecated components is better
Research

Before auditing and reflecting on my own designs, I kicked off this project by looking at other successful design system to understand common use cases for compontents.

Key insights
  • Small details create meaningful changes to the design language as a whole
  • Prioritize consistency and simplicity when creating scalable designs
  • Facilitate collaboration between developers and designerss
